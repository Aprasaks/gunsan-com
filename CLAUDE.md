# Claude 작업 지침

Claude는 군산.com 프로젝트의 개발 파트너이며 항상 `AGENTS.md`를 최우선으로 따릅니다.

## 제품 정의

> 군산.com은 군산을 처음 온 사람이 그대로 따라갈 수 있는 코스와 군산에서 갈 이유가 있는 장소를 먼저 정리하는 여행 큐레이션 플랫폼입니다.

핵심 문장:

> 군산 처음이라면, 이대로만 다녀도 됩니다.

“군산의 모든 것”은 장기 데이터 확장 방향입니다. 메인 브랜드와 첫 사용자 경험은 코스 선택과 필수 선택지 압축입니다.

## 항상 기억할 기준

- `Course`가 중심이고 `Place`는 코스의 재료입니다.
- 많은 장소보다 갈 이유가 있는 3~5개 선택지를 먼저 보여줍니다.
- 메인 홈은 검색·카테고리 목록보다 코스 선택을 먼저 제공합니다.
- 대표 먹거리는 맛의 절대평가가 아니라 상징성, 동선, 여행 경험으로 설명합니다.
- 관광 카페는 군산에서만 갈 이유가 있는 곳을 우선합니다.
- 생활 정보는 장기 확장 영역이며 초기 메인의 중심이 아닙니다.
- 업체 제공, 방문자 확인, 공식 출처, 군산.com 편집 추천을 구분합니다.
- 가짜 수치, 날짜, 실시간 상태, 인기 표현을 만들지 않습니다.
- 외부 리뷰, 사진, 본문, 지도 데이터를 무단 복사하거나 크롤링하지 않습니다.

## 초기 MVP

- 군산 처음 반나절 / 1박 2일 / 2박 3일 코스
- 아이랑 군산 / 비 오는 날 / 선유도 코스
- 군산 필수 5
- 군산 대표 먹거리 5
- 군산다운 카페 5
- 코스별 장소 상세 구조
- 제보하기와 사장님 등록 안내

## 작업 분리

- 방향 문서 정리
- Course 타입 설계
- Course seed 데이터
- 메인 코스 UI
- 장소 상세 보강
- API 연결

위 작업은 각각 별도 이슈와 커밋으로 진행합니다. 문서 작업 중 코드나 데이터를 수정하지 않습니다.

## API 관점

API는 정보를 대량으로 붙이는 수단이 아니라 코스와 필수 장소를 검증하고 보강하는 수단입니다. 공식 행사도 홈에 많이 나열하지 않고 여행 일정에 넣을 수 있는지 확인하는 용도로 사용합니다.

## 기술 규칙

- Next.js 관련 코드 변경 전 `node_modules/next/dist/docs/`의 해당 버전 문서를 읽습니다.
- TypeScript `any`를 사용하지 않습니다.
- `area`, `statuses` 필드명을 유지합니다.
- API 호출은 `lib/` 또는 `lib/api/`로 분리합니다.
- 코드 변경 시 `npm run lint`와 `npm run build`를 실행합니다.
- 문서만 변경하면 `git status`, `git diff`, `git diff --check`로 확인합니다.

## 권한

- 실제 GitHub Issue 번호가 없거나 불명확하면 commit/push하지 않습니다.
- 사용자의 명시적 지시 없이 commit/push하지 않습니다.
- force push하지 않습니다.

세부 기준은 `AGENTS.md`, `docs/WORKFLOW.md`, `docs/SKILL.md`, `docs/DESIGN.md`를 따릅니다.
