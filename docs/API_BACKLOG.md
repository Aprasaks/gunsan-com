# 군산.com API 백로그

## 문서 목적

이 문서는 군산.com에 앞으로 연결할 수 있는 API와 공공데이터 후보를 정리합니다.

군산.com의 목적은 API를 많이 붙이는 것이 아닙니다. 핵심은 **공식 출처 정보**, **업체 제공 정보**, **방문자 확인 정보**를 구분해서 보여주는 것입니다.

API는 사용자가 확인해야 하는 정보를 보조하는 수단입니다. 한 번에 여러 API를 붙이지 않고, 하나의 API는 하나의 작업으로 분리합니다.

## API 연결 원칙

- 하나의 API는 하나의 작업으로 분리합니다.
- API 호출 코드는 컴포넌트 안에 길게 작성하지 않고 `lib/` 또는 `lib/api/`에 분리합니다.
- 실패 상태와 fallback 처리를 반드시 둡니다.
- 실제 값이 없으면 가짜 수치를 만들지 않습니다.
- 공공데이터라도 최신성과 출처를 확인합니다.
- 외부 리뷰, 사진, 본문은 무단 복사하지 않습니다.
- 공식 데이터는 “공식 출처 기준”으로 표현하고, 추천/검증 정보처럼 과장하지 않습니다.
- 실시간성이 있는 데이터는 갱신 주기와 실패 상태를 함께 설계합니다.

| 기능 | 필요한 키 이름 | 필수 여부 | 현재 연결 여부 | 없을 때 fallback | 적용 시점 |
| --- | --- | --- | --- | --- | --- |
| 날씨 | 없음 | 불필요 | Open-Meteo 연결됨 | 군산 날씨 확인 중 | 현재 사용 |
| 기상청 날씨 | DATA_GO_KR_SERVICE_KEY | 공식 전환 시 필요 | 미연결 | 군산 날씨 확인 중 | 나중에 검토 |
| 미세먼지/대기질 | AIRKOREA_SERVICE_KEY 또는 DATA_GO_KR_SERVICE_KEY | 실제 값 표시 시 필요 | 연결 준비됨 | 대기질 확인 중 | 키 발급 후 |

## 1단계: 오늘의 군산

### 날씨

사용 위치:

- Header
- TodayGunsanSection

주의:

- 현재 상태를 짧게 보여주는 용도로 사용합니다.
- API 실패 시 `군산 날씨 확인 중`처럼 안전한 fallback을 사용합니다.
- 실제 값이 없으면 온도, 날씨 상태를 임의로 만들지 않습니다.

### 미세먼지/대기질

사용 위치:

- TodayGunsanSection
- 향후 오늘의 군산 상세 영역

주의:

- 실제 측정소 또는 시도 기준을 확인해야 합니다.
- 측정소 위치와 군산 대표값의 관계를 설명할 수 있어야 합니다.
- 가짜 수치나 임의 등급을 만들지 않습니다.

## 2단계: 생활 정보

### 주차장

사용 위치:

- 자주 확인하는 정보
- 지도에서 보기
- 장소 상세페이지 주변 정보

주의:

- 실시간 주차 가능 여부처럼 표현하지 않습니다.
- 위치, 운영 시간, 요금 같은 기본 정보부터 검토합니다.

### 약국

사용 위치:

- 병원/약국
- 자주 확인하는 정보
- 장소 상세페이지

주의:

- 영업 중 여부를 실시간처럼 확정하지 않습니다.
- 휴일/야간 정보는 출처와 갱신 기준을 확인합니다.

### 병원/의료기관

사용 위치:

- 병원/약국
- 장소 상세페이지

주의:

- 실시간 진료 가능처럼 표현하지 않습니다.
- 진료 과목, 위치, 연락처 등 기본 정보 중심으로 다룹니다.

### 음식점 현황

사용 위치:

- 맛집
- 군산 장소 둘러보기
- 장소 상세페이지

주의:

- 음식점 데이터는 맛집 추천이 아니라 존재와 기본정보 확인용으로 사용합니다.
- 영업 여부, 메뉴, 가격을 임의로 보강하지 않습니다.

### 모범음식점

사용 위치:

- 맛집
- 자주 확인하는 정보
- 장소 상세페이지

주의:

- 공식 선정 정보로 표현합니다.
- “검증된 맛집”처럼 과장하지 않습니다.

### 착한가격업소

사용 위치:

- 생활 정보
- 맛집
- 자주 확인하는 정보

주의:

- 생활 정보 관점으로 표현합니다.
- 현재 지정 상태와 출처 갱신일을 확인합니다.

## 3단계: 여행 정보

### 관광지/TourAPI

사용 위치:

- 관광/여행
- 군산 장소 둘러보기
- 지도에서 보기

주의:

- 장소 DB 구조를 정리한 뒤 연결합니다.
- 외부 설명문을 그대로 복사하지 않습니다.
- 사진 사용권한을 별도로 확인합니다.

### 숙박업소

사용 위치:

- 숙박
- 관광/여행
- 장소 상세페이지

주의:

- 공식 등록 정보로 표현합니다.
- 예약 가능 여부, 가격, 객실 상태를 임의로 만들지 않습니다.

### 문화시설

사용 위치:

- 관광/여행
- 행사·축제
- 지도에서 보기

주의:

- 운영 시간과 휴관일은 최신성 확인이 필요합니다.
- 행사 데이터와 시설 데이터를 섞어 표현하지 않습니다.

### 전통시장

사용 위치:

- 관광/여행
- 생활 정보
- 지도에서 보기

주의:

- 시장 소개, 위치, 운영 정보 중심으로 시작합니다.
- 개별 점포 정보는 별도 구조가 필요합니다.

### 여객선/섬 이동 정보

사용 위치:

- 관광/여행
- 여행 동선
- 자주 확인하는 정보

주의:

- 운항 여부는 최신성 확인이 매우 중요합니다.
- 결항, 시간표, 요금은 공식 출처 기준으로 확인합니다.

### 행사/축제

사용 위치:

- 행사·축제
- TodayGunsanSection
- 관광/여행

주의:

- 공식 출처 확인 전 날짜를 확정하지 않습니다.
- 실제 개최 중, 오늘 진행 중 같은 표현은 데이터 연결 후에만 사용합니다.

## 4단계: 이동 정보

### 버스정류장

사용 위치:

- 지도에서 보기
- 대중교통
- 장소 상세페이지 주변 이동 정보

주의:

- 정류장 위치와 노선 데이터의 출처를 확인합니다.
- 실제 위치 좌표는 지도 기능과 함께 검증합니다.

### 버스도착

사용 위치:

- 대중교통
- 여행 동선

주의:

- 실시간성/fallback 처리가 필수입니다.
- API 실패 시 화면이 깨지지 않아야 합니다.

### 공영자전거

사용 위치:

- 이동 정보
- 여행 동선
- 지도에서 보기

주의:

- 대여 가능 상태를 실시간처럼 표시하려면 API 갱신 기준을 확인합니다.
- 위치 정보는 지도 기능과 분리해서 검토합니다.

### 지도 API

사용 위치:

- 지도에서 보기
- 여행 동선
- 장소 상세페이지 주변 이동 정보

주의:

- 지도 API는 별도 작업으로 분리합니다.
- 실제 좌표와 마커는 검증 후 사용합니다.
- 외부 지도 이미지나 지도 데이터를 무단으로 사용하지 않습니다.

## 5단계: 제안서/행정 데이터

### 관광지점 월별 관광객수

사용 위치:

- 운영자 분석
- 제안서
- 시청/시장 설명 자료
- 향후 통계 페이지

주의:

- 사용자 화면에 바로 노출하기보다 분석/제안서용으로 우선 검토합니다.
- 수치의 기준 기간과 집계 방식을 확인합니다.

### 군산시 현황

사용 위치:

- 운영자 분석
- 제안서
- 시청/시장 설명 자료

주의:

- 최신 기준일을 명확히 표시합니다.
- 사용자에게 직접 보여줄 때는 맥락 설명이 필요합니다.

### 산업단지 정보

사용 위치:

- 운영자 분석
- 생활업체 확장 검토
- 제안서

주의:

- 일반 사용자용 정보와 사업 제안용 정보를 구분합니다.
- 위치/업종/기업 정보의 공개 가능 범위를 확인합니다.

### 홍보포털 사진 현황

사용 위치:

- 제안서
- 향후 이미지 asset 검토
- 시청/시장 설명 자료

주의:

- 이미지 자료는 사용권한 확인이 필수입니다.
- 사용권한이 불확실하면 서비스 화면에 사용하지 않습니다.
