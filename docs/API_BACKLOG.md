# 군산.com API 백로그

## 목적

API는 군산 장소를 많이 자동 수집하거나 메인을 채우는 수단이 아닙니다.

> 군산.com에서 API와 공식 사이트는 코스와 필수 장소를 검증하고 여행 결정을 보강하는 보조 수단입니다.

API 응답은 추천 그 자체가 아닙니다. 군산.com은 공식 정보를 확인한 뒤 코스의 순서, 갈 이유, 대체 장소, 이동 주의사항으로 재편집합니다.

## 연결 원칙

- 하나의 API는 하나의 이슈로 분리합니다.
- API 연결 전에 어떤 코스 판단을 보강하는지 정의합니다.
- 호출 로직은 `lib/` 또는 `lib/api/`에 둡니다.
- 외부 응답은 `unknown`에서 검증합니다.
- 로딩, 실패, fallback을 처리합니다.
- 공식 출처 링크와 마지막 확인일을 함께 관리합니다.
- 실제 값이 없으면 수치, 날짜, 상태를 만들지 않습니다.
- 갱신 주기를 모르면 실시간으로 표현하지 않습니다.
- 외부 본문, 리뷰, 사진을 그대로 복사하지 않습니다.
- API 결과로 맛집 순위나 추천 순서를 자동 생성하지 않습니다.
- 대량 수집보다 수동 편집과 검증 가능한 소량 데이터를 우선합니다.

## API 판단 질문

연결 전 다음 질문에 답해야 합니다.

1. 어떤 Course 또는 CourseStop을 보강하는가?
2. 사용자가 어떤 결정을 더 안전하게 할 수 있는가?
3. 공식성과 갱신 주기는 무엇인가?
4. 실패하면 코스가 잘못 안내되지 않는가?
5. 원문과 이미지를 사용할 권한이 있는가?
6. 마지막 확인일을 저장할 수 있는가?

## 현재 연결 상태

| 기능 | 키 | 상태 | 코스에서의 역할 | 실패 시 |
| --- | --- | --- | --- | --- |
| 날씨 | 없음 | Open-Meteo 연결됨 | 비 오는 날/야외 코스 주의사항 보조 | 확인 중 상태, 임의 값 금지 |
| 기상청 날씨 | ServiceKey | 미연결 | 공식 날씨 전환 후보 | 기존 안전 fallback |
| 대기질 | `AIRKOREA_SERVICE_KEY` | 준비됨 | 아이/야외 코스 주의사항 보조 | 값 생략 또는 확인 중 |
| 지도 | 지도 서비스 키 | 미연결 | 코스 순서와 이동 이해 | 정적 코스 설명 유지 |
| 관광/행사 | `TOUR_API_SERVICE_KEY` | 후보 점검 | 장소/행사 공식 정보 검증 | 수동 검증 콘텐츠 유지 |
| 여객선 | 출처별 확인 | 미연결 | 선유도/섬 코스 이동 검증 | 공식 링크 안내 |

## 우선순위

### 1순위: 코스 운영 검증

#### 공식 운영 시간과 휴관일

목적:

- CourseStop 권장 방문 시간이 실제 운영 조건과 맞는지 확인
- 휴무 시 `fallbackPlace` 제시

주의:

- 업체 제공과 공식 출처를 구분합니다.
- 영업 중을 실시간처럼 단정하지 않습니다.
- 확인일이 오래되면 `확인 필요`로 낮춥니다.

#### 날씨

목적:

- 비 오는 날 코스 진입
- 선유도와 야외 코스 주의사항
- 아이 동반 야외 일정 판단 보조

주의:

- 날씨가 자동으로 코스를 추천하거나 숨기게 만들기 전에 fallback을 설계합니다.
- 예보와 현재 날씨를 구분합니다.

#### 주차

목적:

- 코스 시작 지점과 정류장별 주차 메모
- 주차가 어려운 경우 대체 이동 안내

주의:

- 실시간 주차 가능 대수처럼 표현하지 않습니다.
- 위치, 운영 시간, 요금, 마지막 확인일을 우선합니다.

### 2순위: 이동 검증

#### 지도 API

목적:

- CourseStop 순서 시각화
- 도보/자가용 동선 이해
- 다음 이동 보조

주의:

- 지도는 추천을 만드는 수단이 아니라 이미 편집된 코스를 이해하는 도구입니다.
- 실제 좌표와 마커를 검증합니다.
- 지도 SDK 연결은 별도 작업입니다.

#### 여객선/섬 이동 정보

목적:

- 선유도 및 고군산권 코스의 이동 가능성 확인
- 결항, 시간표, 요금의 공식 출처 안내

주의:

- 최신성이 매우 중요합니다.
- 운항 여부를 캐시된 값만으로 확정하지 않습니다.
- 공식 운항 출처로 이동할 수 있게 합니다.

#### 버스/대중교통

목적:

- 자가용 없는 여행자를 위한 대체 코스 검토
- 정류장과 다음 이동 안내

주의:

- 실시간 도착은 별도 API와 실패 처리가 필요합니다.
- 배차가 드문 구간은 코스 추천 적합성을 별도로 검토합니다.

### 3순위: 필수 장소 검증

#### TourAPI 관광지

목적:

- 필수 장소의 공식 명칭, 주소, 링크, 기본 운영 정보 보강
- 후보 장소 발견이 아니라 이미 편집된 장소 검증을 우선

주의:

- 외부 설명문을 그대로 복사하지 않습니다.
- 이미지 URL이 있어도 사용권한을 확인합니다.
- API에 있다는 이유만으로 코스에 넣지 않습니다.

#### 음식점/모범음식점/착한가격업소

목적:

- 대표 먹거리 장소의 등록 상태와 기본 정보 보강
- 공식 지정 여부를 별도 사실로 표시

주의:

- 공공데이터는 맛의 우수성을 증명하지 않습니다.
- “검증된 맛집”으로 표현하지 않습니다.
- 군산 여행의 상징성과 코스 적합성은 편집 판단으로 분리합니다.

#### 문화시설과 전통시장

목적:

- 근대거리/비 오는 날 코스의 운영 정보 보강
- 휴관일과 체류 가능 시간 확인

주의:

- 시설 정보와 행사 정보를 섞지 않습니다.
- 현재 운영 여부와 마지막 확인일을 관리합니다.

### 4순위: 코스에 포함할 공식 행사

행사 API의 목적은 홈 행사 카드를 많이 채우는 것이 아닙니다.

확인할 것:

- 여행 날짜에 실제 개최되는가?
- 기존 코스에 넣을 가치가 있는가?
- 어느 정류장을 대체하거나 추가하는가?
- 주차와 이동 시간이 달라지는가?
- 공식 출처와 마지막 확인일은 무엇인가?

후보 출처:

- 한국관광공사 TourAPI
- 전국문화축제표준데이터
- 군산문화관광 공식 사이트
- 군산시간여행축제 공식 사이트

상세 기준:

- [EVENT_API_RESEARCH.md](EVENT_API_RESEARCH.md)
- [GUNSAN_OFFICIAL_EVENT_SOURCE_RESEARCH.md](GUNSAN_OFFICIAL_EVENT_SOURCE_RESEARCH.md)

“오늘 진행 중”, “이번 주 인기”, 조회수는 실제 데이터와 검증 없이는 사용하지 않습니다.

### 5순위: 생활 정보 확장

약국, 병원, 생활업체 데이터는 장기 확장 영역입니다. 초기 코스 중심 메인의 핵심 API 우선순위가 아닙니다.

필요 시 다음 상황에서 별도 검토합니다.

- 여행 중 긴급 의료 정보
- 코스 인근 화장실/편의시설
- 장기 체류자의 생활 편의

의료 정보는 실시간 진료 가능처럼 단정하지 않고 공식 출처와 연락처를 우선합니다.

## 행사 API 사용 기준

행사를 Course에 연결할 때 필요한 최소 필드:

- 공식 행사명
- 시작일과 종료일
- 장소
- 공식 상세 URL
- 출처명
- 마지막 확인일
- Course 연결 이유
- 대체할 CourseStop 또는 추가 위치
- 이동/주차 주의사항

이미지는 필수 필드가 아닙니다. 이미지가 없다는 이유로 비공식 이미지를 사용하지 않습니다.

## fallback 원칙

- API 실패 시 코스 전체가 깨지지 않아야 합니다.
- 검증이 필요한 정류장은 상태를 낮추거나 대체 장소를 안내합니다.
- 오래된 행사 정보는 숨기거나 공식 링크 확인을 안내합니다.
- 날씨/운항처럼 변동성이 큰 정보는 최종 확인 행동을 제공합니다.
- seed fallback에도 가짜 날짜와 수치를 넣지 않습니다.

## 분석용 데이터

관광지점 방문자 수, 군산시 현황 등은 운영자 분석과 제안서에 사용할 수 있습니다.

- 사용자 추천 순위로 직접 변환하지 않습니다.
- 집계 기간과 방식 표시
- 관광객 수가 장소 품질이나 코스 적합성을 의미하지 않음을 명시
- 메인 사용자 화면보다 내부 분석을 우선
