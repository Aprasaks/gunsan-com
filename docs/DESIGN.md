# 군산.com 디자인 규칙 (DESIGN.md)

## 디자인 정체성

군산.com은 **"군산의 모든 것"** 을 목표로 하는 지역 정보 웹사이트입니다.

군산의 맛집, 카페, 병원, 숙박, 생활업체, 관광지, 행사 정보를 한곳에서 확인할 수 있는 구조를 지향합니다.

다만 초기 MVP에서는 모든 정보를 한 번에 보여주려 하지 않습니다.
v0.1에서는 장소 정보, 업체 제공 정보, 방문자 확인 정보, 제보 구조를 위한 기본 UI부터 작게 시작합니다.

군산.com은 군산에 사는 사람과 군산에 오는 사람이 모두 사용할 수 있는 **군산 대표 정보 포털**을 지향합니다.

- 메인 홈은 단순한 검색/필터 화면이 아니라, 군산의 맛집, 카페, 병원, 생활업체, 숙박, 관광/여행, 행사/축제로 들어가는 대표 입구 역할을 합니다.
- 장소를 깊게 탐색하는 필터 중심 구조는 /places 페이지에서 담당합니다.
- 군산.com은 생활 정보와 여행 정보를 함께 다루되, 정보의 출처와 검증 상태를 분리해서 보여줍니다.

군산.com은 Dechive의 검증 철학을 지역 생활 정보에 적용하되, 사용자가 처음 느끼는 인상은 밝고 명확한 지역 정보 포털이어야 합니다.

Dechive처럼 고딕·잡지형 디자인이 아닙니다.
군산.com은 누구나 쉽게 이해하고, 빠르게 찾고, 비교할 수 있는 정보 중심 사이트입니다.

**디자인 키워드:**
명확함 · 신뢰감 · 빠른 탐색 · 쉬운 비교 · 정보 우선 · 광고 느낌 최소화

## 디자인 목표

군산.com의 디자인은 다음 질문에 답해야 합니다.

- 여기가 군산 정보를 찾는 사이트라는 것이 바로 보이는가?
- 군산에 사는 사람과 여행 오는 사람 모두 자신에게 필요한 정보로 빠르게 진입할 수 있는가?
- 사용자가 원하는 정보를 빠르게 검색할 수 있는가?
- 장소 정보와 검증 상태가 한눈에 보이는가?
- 업체 제공 정보와 방문자 확인 정보가 구분되는가?
- 광고 사이트처럼 보이지 않는가?
- 데스크탑에서 넓게 비교하고 탐색할 수 있는가?
- 모바일에서도 정보가 깨지지 않고 자연스럽게 쌓이는가?

## 핵심 브랜드 문장

군산.com의 가장 큰 브랜드 방향은 다음 문장입니다.

> 군산.com = 군산의 모든 것

### 보조 브랜드 문장

- 군산에 산다면, 군산.com.
- 군산에 여행 가고 싶다면, 군산.com.
- 사는 사람도, 오는 사람도 군산.com에서 시작합니다.

사용자에게 보이는 메인 메시지는 쉽고 단순해야 합니다.

### 메인 카피

| 구역             | 문구                                                                                                               |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ |
| 큰 문구          | 군산의 모든 것, 군산.com에서 시작하세요                                                                            |
| 작은 문구        | 밥 먹을 곳, 쉬어갈 카페, 급하게 찾는 병원, 주차 되는 곳까지. 군산에서 필요한 정보를 하나씩 모으고 있습니다.      |
| 검증 보조 문구   | 사장님이 알려준 정보와 실제 다녀온 사람이 확인한 정보를 나눠서 보여줍니다.                                         |
| 검색 placeholder | 맛집, 병원, 카페, 행사 등 군산의 정보를 검색해 보세요                                                             |

### 사용 가능한 보조 문장

- 군산에서 어디 갈지 고민될 때, 군산.com.
- 군산의 맛집, 생활업체, 행사 정보를 한곳에서 확인하세요.
- 사장님이 알려준 정보와 실제 다녀온 사람이 확인한 정보를 나누어 보여드립니다.
- 정보가 달라졌다면 제보해주세요.

## 메인 홈과 장소 탐색 페이지 역할 분리

### 메인 홈

메인 홈은 군산 대표 포털형 입구입니다.

역할:
- 군산에 사는 사람과 군산에 오는 사람이 모두 처음 들어오는 화면
- 군산의 주요 정보 카테고리로 빠르게 이동하는 화면
- 오늘의 군산, 행사/축제, 지도 보기, 동네별 보기 같은 확장 가능성을 보여주는 화면
- 군산.com이 "군산의 모든 것"을 담는 사이트라는 인상을 주는 화면

메인 홈에서 우선 보여줄 수 있는 영역:
- 대표 히어로
- 통합 검색
- 주요 카테고리 진입
- 지금 많이 찾는 정보
- 군산 장소 둘러보기
- 오늘의 군산
- 행사/축제 예정 영역
- 지도에서 보기 예정 영역
- 동네별 보기 예정 영역
- 정보 제보하기

### /places

/places는 필터 중심 장소 탐색 페이지입니다.

역할:
- 장소 목록 탐색
- 카테고리/동네/상태 필터
- 장소 카드 비교
- 검증 상태 확인
- 장소 상세 페이지 진입

/places에서는 기존 데스크탑 3단 레이아웃을 사용할 수 있습니다.

## 레이아웃 원칙

군산.com은 데스크탑 정보 탐색을 기본으로 합니다.

모바일에서도 잘 보여야 하지만, 모바일 앱처럼 보이는 구조를 우선하지 않습니다.
기본은 웹사이트이며, 모바일은 반응형으로 자연스럽게 대응합니다.

### 기본 원칙

- 메인 홈은 포털형 섹션 레이아웃을 기본으로 한다.
- /places 같은 탐색 페이지는 데스크탑 3단 레이아웃을 사용할 수 있다.
- 메인 홈에서는 필터 패널보다 대표 히어로, 카테고리, 추천/인기 정보, 행사/지도/동네 섹션이 먼저 보인다.
- 데스크탑에서는 넓은 화면을 활용해 여러 섹션을 한눈에 보여준다.
- 모바일에서는 섹션이 자연스럽게 아래로 쌓인다.
- 앱처럼 보이는 모바일 전용 UI 금지
- 정보 비교가 쉬운 카드형 레이아웃 사용

---

## 데스크탑 3단 레이아웃

데스크탑 3단 레이아웃은 /places 같은 장소 탐색 페이지에서 사용합니다.

```txt
[좌: 필터 패널] | [중: 장소 목록] | [우: 사이드바 / 미리보기]
```

### 좌측 영역

역할:

- 카테고리 필터
- 태그 필터
- 검증 상태 필터
- 동네/지역 필터
- 빠른 조건 필터

예시:

- 맛집
- 카페
- 네일/뷰티
- 병원/약국
- 생활업체
- 주차 가능
- 오늘 영업
- 방문자 사진 있음
- 확인 필요

### 중앙 영역

역할:

- 장소 카드 목록
- 검색 결과
- 추천 장소
- 최근 확인된 장소

장소 카드는 정보 우선으로 구성합니다.

### 우측 영역

역할:

- 선택된 장소 요약
- 자주 찾는 질문
- 최근 확인 정보
- 정보 제보 CTA
- 군산.com의 검증 원칙 안내

---

## 모바일 레이아웃 원칙

모바일에서는 데스크탑 3단 구조를 억지로 유지하지 않습니다.

다음 순서로 자연스럽게 쌓습니다.

1. 헤더
2. 검색 영역
3. 빠른 질문 칩
4. 카테고리 탭
5. 필터 토글
6. 장소 카드 목록
7. 보조 정보
8. 제보 CTA

### 모바일 금지 사항

- 앱처럼 보이는 하단 탭바
- 지나치게 큰 CTA 버튼 반복
- 화면을 가리는 과한 모달
- 검색보다 이미지가 먼저 보이는 구조
- 정보보다 감성 문구가 더 커지는 구조

## 메인 홈 섹션 디자인 규칙

### 대표 히어로

- 군산 대표 사이트라는 인상이 첫 화면에서 바로 느껴져야 한다.
- 군산의 풍경, 장소, 생활 정보를 암시하는 비주얼 영역을 둘 수 있다.
- 추후 새만금, 금강, 은파, 근대거리 이미지/GIF/짧은 영상이 들어갈 수 있는 구조를 고려한다.
- 단, 실제 영상/GIF 구현은 별도 이슈로 분리한다.
- 검색창은 비주얼보다 더 중요한 기능 진입점으로 보여야 한다.

### 주요 카테고리 진입

메인 홈에서는 v0.1 실제 장소 카테고리뿐 아니라 장기 확장 카테고리도 "준비 중" 상태로 보여줄 수 있다.

표시 가능한 카테고리:
- 맛집
- 카페
- 병원/약국
- 생활업체
- 네일/뷰티
- 숙박
- 관광/여행
- 행사/축제
- 주차
- 문화/여가

주의:
- v0.1 실제 데이터 모델의 PlaceCategory와 혼동하지 않는다.
- 숙박, 관광/여행, 행사/축제 등은 v0.1에서 실제 데이터 필터로 구현하지 않는다.
- 준비 중 카테고리는 준비 중임을 과하지 않게 표시한다.

### 지금 많이 찾는 정보

- 군산 사람들이 자주 찾는 생활 정보와 여행자가 자주 찾는 정보를 함께 보여준다.
- 예: 오늘 여는 병원, 주차장 정보, 군산사랑상품권, 은파호수공원, 새만금방조제, 행사 정보
- 실제 데이터가 없을 경우 구조 확인용 placeholder로 표시한다.

### 군산 장소 둘러보기

- 대표 장소 카드 또는 장소 미리보기 영역이다.
- 장소 카드는 정보 우선으로 구성한다.
- 사진이 들어갈 수 있지만, 사진이 검증 상태보다 더 중요하게 보이면 안 된다.

### 오늘의 군산

- 날씨, 미세먼지, 일몰 시간, 오늘의 행사 등으로 확장 가능한 영역이다.
- v0.1에서는 실제 API 연결 없이 정적 UI 또는 준비 중 안내만 허용한다.

### 행사/축제

- 군산 여행자에게 중요한 섹션이다.
- v0.1에서는 실제 행사 데이터 연결 없이 예정 영역이나 placeholder로만 표시할 수 있다.
- 공식 출처 기반으로 정리한다는 방향을 유지한다.

### 지도에서 보기

- 군산 장소를 지도에서 볼 수 있다는 확장 가능성을 보여준다.
- v0.1에서는 실제 지도 API나 좌표 기반 마커를 구현하지 않는다.
- "지도에서 보기 준비 중" 또는 "지도 열기" UI는 별도 이슈에서 다룬다.

### 동네별 보기

- 월명동, 수송동, 나운동, 미장동, 소룡동 등 동네별 탐색으로 확장할 수 있다.
- v0.1에서는 실제 동네 필터 기능 없이 진입 UI 또는 placeholder만 둘 수 있다.

## 색상 시스템

군산.com은 밝고 명확해야 합니다.

| 용도                  | 방향                                                |
| --------------------- | --------------------------------------------------- |
| 전체 배경             | 흰색 또는 아주 연한 회색 (`slate-50` 수준)          |
| 카드 배경             | 흰색                                                |
| 본문 텍스트           | 진한 slate 계열 (`slate-800` ~ `slate-900`)         |
| 보조 텍스트           | `slate-500` ~ `slate-600`                           |
| 경계선                | `slate-200` 수준                                    |
| 포인트 컬러           | 군산 바다 느낌의 파랑/청록 (`sky-500` ~ `teal-500`) |
| 확인 상태 배지        | 초록 (`green-500` ~ `emerald-500`)                  |
| 미확인 / 확인 필요    | 회색 또는 노랑 (`slate-400`, `yellow-400`)          |
| 현장 차이 제보 있음   | 주황 (`orange-500`)                                 |
| 폐업/이전 확인 필요   | 빨강 (`red-500`)                                    |
| 업체 제공 정보 영역   | 파랑 계열 구분선 또는 배경 틴트                     |
| 방문자 확인 정보 영역 | 초록 계열 구분선 또는 배경 틴트                     |

구체적인 색상 값은 Tailwind CSS 유틸리티로 구현합니다.

디자인 토큰이 필요할 경우 추후 `tailwind.config.ts`에 추가합니다.

## 검증 상태 배지 (StatusBadge)

`VerificationStatus` 값에 따라 색상과 레이블을 다르게 표시합니다.

| 상태                | 배지 색      | 의미                                |
| ------------------- | ------------ | ----------------------------------- |
| 최근 확인됨         | 초록         | 방문자 또는 관리자가 최근 직접 확인 |
| 방문자 후기         | 초록 연한 톤 | 방문자 후기 존재                    |
| 방문자 사진         | 초록 연한 톤 | 방문자 사진 존재                    |
| 업체 제공           | 파랑         | 사장님 제공 정보                    |
| 정보 수집됨         | 회색         | 기본 정보만 있음                    |
| 확인 필요           | 노랑         | 정보 최신성 불확실                  |
| 현장 차이 제보 있음 | 주황         | 온라인 정보와 실제 차이 제보 있음   |
| 폐업/이전 확인 필요 | 빨강         | 폐업 또는 이전 가능성               |

### 배지 디자인 원칙

- 너무 화려하게 만들지 않습니다.
- 상태가 한눈에 보이게 합니다.
- 색상만으로 의미를 전달하지 않고 텍스트를 함께 표시합니다.
- "최근 확인됨"과 "업체 제공"을 절대 같은 의미처럼 보이게 하지 않습니다.
- 확인되지 않은 정보를 긍정적으로 보이게 꾸미지 않습니다.

## 타이포그래피

한글 가독성을 최우선으로 합니다.

### 기본 원칙

- 제목: 굵고 명확하게
- 본문: 읽기 쉬운 중간 굵기
- 정보성 텍스트: 작더라도 판독 가능한 크기 유지
- 최소 본문 크기: `text-sm`
- 과도한 이탤릭체 금지
- 감성적 손글씨 서체 금지
- 장식용 폰트 남용 금지

| ----------- | ----------------------- |
| 페이지 제목 | 크고 명확하게           |

## SearchHero

### 역할

메인 홈에서 군산 대표 포털이라는 인상을 가장 먼저 전달합니다.

### 포함 요소

- 큰 문구
- 작은 문구
- 검증 보조 문구
- 검색창 또는 검색 진입 UI
- CTA 버튼

### 디자인 원칙

- 검색창은 항상 핵심 진입점으로 유지합니다.
- 통합 검색과 군산 비주얼 카드가 함께 있을 수 있습니다.
- 추후 이미지/GIF/짧은 영상이 들어갈 수 있는 구조를 고려합니다.
- 배경을 지나치게 화려하게 꾸미지 않습니다.
- "군산의 모든 것"이라는 인상이 바로 전달되어야 합니다.
- 과한 그라디언트나 SaaS 랜딩 페이지 느낌을 피합니다.
- 비주얼이 검색 기능을 가리면 안 됩니다.

## 정보 분리 원칙

업체 제공 정보와 방문자 확인 정보는 UI에서 반드시 시각적으로 구분합니다.

### 업체 제공 정보

표시 예시:

- 섹션 제목: `업체 제공 정보`
- 색상 방향: 파랑 계열 구분선 또는 배경 틴트
- 의미: 사장님 또는 업체가 제공한 정보

### 방문자 확인 정보

표시 예시:

- 섹션 제목: `방문자 확인 정보`
- 색상 방향: 초록 계열 구분선 또는 배경 틴트
- 의미: 실제 방문자 제보 또는 확인 정보

### 확인 필요 정보

표시 예시:

- 레이블: `확인 필요`
- 색상 방향: 노랑 또는 회색
- 의미: 최신성 또는 정확성이 아직 불확실한 정보

### 금지

- 하나의 카드나 섹션 안에 출처가 다른 정보를 섞지 않습니다.
- 업체 제공 사진을 방문자 사진처럼 보이게 하지 않습니다.
- 방문자 확인이 없는 정보를 최근 확인된 것처럼 표시하지 않습니다.

## 최종 디자인 기준

군산.com은 예쁜 사이트보다 **쓸모 있는 사이트**여야 합니다.

사용자가 들어왔을 때 가장 먼저 느껴야 하는 것은 다음입니다.

- 군산 정보를 찾는 곳이구나
- 군산에 사는 사람도, 군산에 여행 오는 사람도 쓸 수 있구나
- 검색하기 쉽구나
- 장소를 비교하기 쉽구나
- 정보 상태가 보이는구나
- 업체 말과 방문자 확인이 구분되는구나
- 내가 아는 정보가 틀리면 제보할 수 있구나

군산.com의 큰 비전은 "군산의 모든 것"입니다.
하지만 디자인은 항상 정보 우선, 신뢰 우선, 빠른 탐색 우선으로 구성합니다.
