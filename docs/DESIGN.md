# 군산.com 디자인 규칙 (DESIGN.md)

## 디자인 정체성

군산.com은 Dechive의 검증 철학을 지역 생활 정보에 적용한 프로젝트입니다.
Dechive처럼 고딕·잡지형 디자인이 아닙니다. 밝고 명확하고 누구나 이해하기 쉬운 지역 정보 검색 사이트여야 합니다.

**디자인 키워드:** 명확함 · 신뢰감 · 빠른 탐색 · 쉬운 비교 · 정보 우선 · 광고 느낌 최소화

---

## 레이아웃 원칙

- 기본: 데스크탑 정보 탐색 최적화 (3단 레이아웃)
- 반응형: 모바일에서도 보기 좋게 대응
- 앱처럼 보이는 모바일 전용 UI 금지

### 데스크탑 3단 레이아웃

```
[좌: 필터 패널] | [중: 장소 목록] | [우: 사이드바 / 미리보기]
```

- 좌측: 카테고리 필터, 태그 필터
- 중앙: 장소 카드 목록
- 우측: 선택된 장소 요약, 빠른 정보

## 색상 시스템

| 용도 | 방향 |
|---|---|
| 배경 | 흰색 또는 아주 연한 회색 (`slate-50` 수준) |
| 본문 텍스트 | 진한 slate 계열 (`slate-800` ~ `slate-900`) |
| 포인트 컬러 | 군산 바다 느낌의 파랑/청록 (`sky-500` ~ `teal-500` 참고) |
| 확인 상태 배지 | 초록 (`green-500` ~ `emerald-500`) |
| 미확인 / 확인 필요 | 회색 또는 노랑 (`slate-400`, `yellow-400`) |
| 현장 차이 제보 있음 | 주황 (`orange-500`) |
| 폐업/이전 확인 필요 | 빨강 (`red-500`) |
| 업체 제공 정보 영역 | 파랑 계열 (구분선 또는 배경 틴트) |
| 방문자 확인 정보 영역 | 초록 계열 (구분선 또는 배경 틴트) |

구체적인 색상 값은 Tailwind 유틸리티로 구현하며, 디자인 토큰이 필요할 경우 추후 `tailwind.config.ts`에 추가합니다.

## 검증 상태 배지 (StatusBadge)

`VerificationStatus` 값에 따라 색상과 레이블을 다르게 표시합니다.

| 상태 | 배지 색 | 의미 |
|---|---|---|
| 최근 확인됨 | 초록 | 방문자 또는 관리자가 최근 직접 확인 |
| 방문자 후기 | 초록 (연한) | 방문자 후기 존재 |
| 방문자 사진 | 초록 (연한) | 방문자 사진 존재 |
| 업체 제공 | 파랑 | 사장님 제공 정보 |
| 정보 수집됨 | 회색 | 기본 정보만 있음 |
| 확인 필요 | 노랑 | 정보 최신성 불확실 |
| 현장 차이 제보 있음 | 주황 | 온라인 정보와 실제 차이 제보 있음 |
| 폐업/이전 확인 필요 | 빨강 | 폐업 또는 이전 가능성 |

## 타이포그래피

- 한글 가독성 우선
- 제목: 굵고 명확하게
- 본문: 읽기 쉬운 중간 굵기
- 정보성 텍스트: 작더라도 판독 가능한 크기 유지 (최소 `text-sm`)
- 감성적 손글씨 서체, 과도한 이탤릭체 금지

## 컴포넌트 디자인 규칙

### SearchHero

- 메인 검색 영역
- 배경을 지나치게 화려하게 꾸미지 않음
- 검색창이 가장 눈에 띄어야 함

### QuickQuestionChips

- 작고 명확한 칩 형태
- 클릭하면 해당 태그로 필터링
- 칩 레이블: `남자 네일 가능한 곳` 등 실제 질문형

### PlaceCard

- 장소명은 크고 굵게
- 상태 배지는 카드 상단 또는 제목 옆에 배치
- 후기 수와 최근 확인일은 작게, 하단에 배치
- 카드 전체가 클릭 가능한 링크
- 광고처럼 보이는 강조 효과 금지

### FilterPanel

- 좌측 고정 패널 (데스크탑)
- 모바일에서는 드로어 또는 상단 토글로 접힘
- 카테고리 / 태그 / 검증 상태 필터

### RightSidebar

- 선택된 장소의 핵심 정보 미리보기
- 빠른 질문 응답형 정보 표시 (예: "주차 가능한가? → 가능 (방문자 확인)")

## 금지 디자인 패턴

- 어두운 고딕 배경
- 잡지형 블로그 레이아웃
- 감성 포트폴리오 스타일
- 스타트업 SaaS 랜딩 페이지 스타일
- 광고 배너 영역
- 앱처럼 보이는 하단 탭바
- 모바일 전용 UI (데스크탑 경험 저하)
- 화려한 그라디언트 헤더
- 카드에 별점 5개 시각화 (순위 사이트처럼 보임)

## 메인 페이지 카피라이팅

| 구역 | 문구 |
|---|---|
| 큰 문구 | 군산에서 매번 다시 물어봐야 했던 정보를 확인하세요 |
| 작은 문구 | 검색해도 애매한 군산 생활 정보를 한곳에 모아 검증합니다. |
| 검색 placeholder | 남자 네일, 일요일 병원, 주차 카페 |

## 정보 분리 원칙 (UI 반영)

업체 제공 정보와 방문자 확인 정보는 UI에서 반드시 시각적으로 구분합니다.

- 섹션 제목으로 구분: "업체 제공 정보" / "방문자 확인 정보"
- 색상 틴트 또는 구분선으로 시각적 차이 부여
- 하나의 카드나 섹션 안에 두 정보를 섞지 않음
- 확인되지 않은 정보에는 "확인 필요" 레이블 표시
