# 군산.com 디자인 기준

## 디자인 목표

군산.com의 첫 화면은 정보 포털이나 장소 목록이 아니라 **여행 결정 화면**이어야 합니다.

사용자는 첫 화면에서 다음을 느껴야 합니다.

- 군산이 처음이어도 그대로 따라갈 수 있다.
- 장소를 수십 개 비교하지 않아도 된다.
- 내 일정과 상황에 맞는 코스를 바로 고를 수 있다.
- 왜 가는지, 얼마나 머무는지, 다음에 어디로 가는지 알 수 있다.

핵심 디자인 문장:

> 군산 처음이라면, 이대로만 다녀도 됩니다.

“군산의 모든 것”과 생활·관광 포털은 장기 확장 맥락입니다. 첫 화면의 인상과 정보 구조를 결정하는 기준으로 사용하지 않습니다.

## 디자인 키워드

결정 우선 · 선택지 압축 · 군산다운 경험 · 동선 중심 · 신뢰 · 넓은 여백 · 명확한 순서 · 광고 느낌 최소화

## 첫 행동

메인 홈의 첫 행동은 검색이나 카테고리 탐색이 아니라 코스 선택입니다.

권장 우선순위:

1. 군산 처음 반나절
2. 군산 처음 1박 2일
3. 군산 처음 2박 3일
4. 아이랑 군산
5. 비 오는 날 군산
6. 선유도

전체 장소 검색은 이 선택지 다음에 둡니다.

## 메인 카피

| 역할 | 문구 |
| --- | --- |
| 큰 문구 | 군산 처음이라면, 이대로만 다녀도 됩니다. |
| 보조 문구 | 맛집, 카페, 관광지 고르느라 헤매지 않게 군산에서 가야 할 곳만 코스로 정리했습니다. |
| 검색 placeholder | 예: 이성당 다음 코스, 아이랑 군산, 비 오는 날 군산, 선유도 가는 길 |

과장된 관광 홍보 문구, “실시간 인기”, “최고의 맛집”, “모든 정보를 한곳에” 같은 표현은 사용하지 않습니다.

## 메인 홈 권장 순서

1. 투명 헤더
2. full-width 파노라마 히어로
3. 핵심 카피와 코스 선택 CTA
4. 군산 처음 코스 선택
5. 군산 필수 5
6. 군산 대표 먹거리 5
7. 군산다운 카페 5
8. 아이/날씨/주차 등 목적별 코스
9. 선유도 및 지도 진입
10. 전체 장소 검색

카테고리 카드 묶음은 메인의 핵심이 아닙니다. 필요하면 하단 탐색 또는 전체 장소 페이지의 보조 내비게이션으로 사용합니다.

## 헤더

상단 메뉴의 문서상 방향:

- 코스
- 필수장소
- 대표먹거리
- 군산다운카페
- 선유도
- 지도

원칙:

- 히어로 위 투명 헤더를 사용할 수 있습니다.
- 메뉴를 2단 포털 구조로 과밀하게 만들지 않습니다.
- 코스 선택을 방해하는 날씨, 생활업체, 병원/약국 메뉴를 최상단에 나열하지 않습니다.
- 제보하기와 사장님 등록은 보조 유틸리티로 둡니다.
- 기능이 없으면 “준비 중”을 반복하지 말고 안전한 링크 또는 조용한 비활성 상태로 처리합니다.

## 히어로

비짓제주 레퍼런스의 넓은 배경, 투명 헤더, 검색 오버레이 구조는 사용할 수 있습니다. 다만 목적은 관광 홍보가 아니라 여행 결정을 줄이는 것입니다.

원칙:

- full-width 파노라마 이미지 사용
- 텍스트와 컨트롤은 내부 max-width 컨테이너에 배치
- 첫 문구는 “군산 처음이라면, 이대로만 다녀도 됩니다.” 방향
- 검색창보다 반나절/1박 2일/2박 3일 코스 선택을 먼저 또는 같은 위계로 제공
- 이미지 위 문구의 명도 대비 확보
- 실제 군산 장소 사진은 권한과 출처 확인
- 실제 장소처럼 오해되는 허위 AI 이미지를 사용하지 않음

## 코스 선택 UI

코스 카드는 장소 카드보다 먼저 보여야 합니다.

각 코스 카드의 우선 정보:

1. 코스명
2. 소요 시간 또는 여행 일수
3. 누구를 위한 코스인지
4. 핵심 정류장 3~5개
5. 이동 방식
6. 코스 선택 CTA

좋은 예:

- 군산 처음 반나절
- 처음 와도 놓치지 않는 1박 2일
- 아이랑 천천히 도는 군산
- 비가 와도 괜찮은 실내 중심 코스
- 선유도 가는 날

피해야 할 예:

- 장소 20개를 한 카드에 나열
- 검증되지 않은 “인기 1위”
- 실제 데이터 없는 예상 비용이나 소요 시간
- 광고 장소를 핵심 정류장처럼 위장

## 코스 상세 화면

코스 상세는 여행 순서를 한눈에 이해하게 해야 합니다.

권장 구조:

1. 코스 요약
2. 총 예상 시간과 이동 방식
3. 추천 대상 / 비추천 대상
4. 순서가 있는 정류장
5. 각 정류장의 `왜 여기인가`
6. 권장 체류 시간
7. 다음 이동 안내
8. 주차와 날씨 메모
9. 대체 정류장
10. 공식 출처와 마지막 확인일

시각적 타임라인을 사용할 수 있지만 실제 이동 시간과 거리처럼 보이는 값은 검증된 경우에만 표시합니다.

## 군산 필수 5

- 한일옥, 이성당, 초원사진관, 근대거리, 선유도처럼 군산 여행의 상징이 된 장소를 맥락과 함께 보여줍니다.
- “반드시 최고”가 아니라 “왜 군산 여행에서 반복해 선택되는가”를 설명합니다.
- 각 항목에 다음 코스와 대체 장소를 연결합니다.
- 카드 수는 원칙적으로 5개 안팎으로 제한합니다.

## 대표 먹거리

맛을 절대평가하는 별점·랭킹 UI를 사용하지 않습니다.

장소 카드 우선 정보:

- 군산 여행에서 들르는 이유
- 대표 경험 또는 상징
- 어느 코스에 넣기 좋은지
- 권장 방문 시간
- 대기/주차 등 결정에 필요한 조건
- 호불호 또는 확인 필요 정보

“최고의 맛집”, “검증된 맛집”, “현지인 1위” 같은 문구는 근거 없이 사용하지 않습니다.

## 군산다운 카페

메인에서는 체인점이 아니라 군산에서 갈 이유가 있는 카페를 우선합니다.

분류 예:

- 근대거리 산책 뒤 들를 카페
- 선유도 가는 길에 들를 카페
- 군산 바다가 보이는 카페
- 오래된 건물의 분위기를 살린 카페
- 사진 남기기 좋은 카페
- 아이랑 쉬기 좋은 카페
- 비 오는 날 오래 머물기 좋은 카페

체인 카페는 필요하면 “근처에서 잠깐 쉬기” 같은 생활 편의 정보로 분리합니다.

## 목적별 선택

긴 설명 카드보다 짧은 선택 chip 또는 compact card를 사용할 수 있습니다.

- 아이랑
- 비 오는 날
- 사진 찍는 곳
- 주차 편한 코스
- 많이 걷지 않는 코스
- 대중교통 코스

실제 검색량처럼 보이는 “지금 많이 찾는”, 조회수, 실시간 인기 표현은 쓰지 않습니다.

## Place 카드

Place는 Course를 보강하는 역할입니다.

우선 정보:

1. 장소명
2. 왜 가는가
3. 포함 코스
4. 권장 체류 시간
5. 다음 장소
6. 주차/아이/비 오는 날 적합성
7. 검증 상태

별점, 과장된 추천 배지, 광고형 강조 효과를 사용하지 않습니다.

## 지도

지도는 코스의 동선을 이해하는 보조 도구입니다.

- 코스 순서와 이동 맥락을 우선합니다.
- 지도 SDK 연결 전 실제 지도처럼 보이는 확대/축소, 현재 위치, 좌표 마커 UI를 만들지 않습니다.
- 외부 지도 이미지를 무단 사용하지 않습니다.
- 지도 연결은 별도 이슈입니다.

## 행사·축제

행사는 별도 목록을 많이 노출하기보다 해당 여행 날짜와 코스에 넣을 수 있는 공식 일정인지 알려주는 보조 콘텐츠입니다.

- 공식 출처와 마지막 확인일 표시
- 실제 날짜를 확인하지 않은 행사 노출 금지
- “오늘 진행 중”은 실제 날짜와 운영 상태 확인 후에만 표시
- 코스에 포함할 경우 기존 정류장을 대체하는지, 이동에 어떤 영향이 있는지 설명
- 외부 포스터와 이미지는 사용권한 확인

## 데스크탑

- 넓은 히어로와 투명 헤더를 사용할 수 있습니다.
- 첫 화면에서 코스 선택이 명확하게 보여야 합니다.
- 여러 카테고리와 정보 카드를 한 화면에 채우는 포털형 밀도를 목표로 하지 않습니다.
- 3~5개의 명확한 선택과 충분한 여백을 우선합니다.

## 모바일

모바일 첫 화면은 탐색 화면이 아니라 결정 화면입니다.

첫 노출 권장 범위:

1. 핵심 문구
2. 검색 진입
3. 코스 선택
4. 군산 필수 5 일부

원칙:

- 긴 스크롤 금지
- 전체 맛집·카페·관광지 목록을 메인에 길게 나열하지 않음
- 카드 하나가 화면 대부분을 차지하지 않게 함
- 코스 선택은 2열 compact grid 또는 짧은 가로 스크롤 사용 가능
- CTA를 반복하지 않음

## 색상과 타이포그래피

- 밝고 신뢰감 있는 흰색/슬레이트 배경
- 군산 바다를 연상하는 파랑·청록을 포인트로 사용
- 코스 CTA는 한 가지 강조색으로 제한
- 가독성 좋은 한글 산세리프 사용
- 제목은 굵고 짧게, 설명은 선택에 필요한 내용만 작성
- 감성 손글씨, 과한 그라디언트, 광고형 원색 버튼 남발 금지

## 검증 상태와 출처

UI에서 다음을 구분합니다.

- 공식 출처
- 군산.com 편집 추천
- 업체 제공 정보
- 방문자 확인 정보
- 확인 필요
- 현장 차이 제보

업체 제공 정보와 방문자 확인 정보를 하나의 배지나 섹션으로 섞지 않습니다. 편집 추천도 공식 인증처럼 보이게 하지 않습니다.

## 사업화 UI

군산.com은 가게 목록 광고판이 아닙니다.

허용 가능한 맥락:

- 선유도 가는 길에 들를 카페
- 비 오는 날 코스의 실내 공간
- 아이랑 코스의 식사 장소
- 근대거리 산책 후 들를 카페

유료 입점은 반드시 표시하고 편집 추천, 공식 정보, 검증 상태와 분리합니다. 결제 여부가 추천 순서와 검증 상태를 결정해서는 안 됩니다.

## 금지 디자인 패턴

- 생활·관광 카테고리를 같은 무게로 메인에 나열
- 카테고리 카드가 첫 행동이 되는 홈
- 장소 수와 정보량을 가치처럼 강조
- 모바일 메인의 긴 장소 목록
- 별점과 맛집 순위 중심 UI
- 가짜 실시간, 조회수, 인기 배지
- 광고와 편집 추천의 혼합
- 코스보다 이미지나 검색창이 더 중요한 화면
- 앱처럼 보이는 하단 탭 중심 구조

## 최종 판단 기준

디자인을 검토할 때 다음 질문에 답합니다.

- 처음 온 사람이 첫 화면에서 코스를 고를 수 있는가?
- 선택지가 3~5개 수준으로 압축되어 있는가?
- 각 장소의 갈 이유와 다음 이동이 보이는가?
- 검색하지 않아도 대표 여행을 시작할 수 있는가?
- 모바일 첫 화면이 결정 화면으로 작동하는가?
- 광고, 공식 정보, 편집 추천, 방문자 확인이 구분되는가?
