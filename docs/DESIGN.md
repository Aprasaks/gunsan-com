# 군산.com 디자인 규칙 (DESIGN.md)

## 디자인 정체성

군산.com은 **“군산의 모든 것”** 을 목표로 하는 지역 정보 웹사이트입니다.

군산의 맛집, 카페, 병원, 숙박, 생활업체, 관광지, 행사 정보를 한곳에서 확인할 수 있는 구조를 지향합니다.

다만 초기 MVP에서는 모든 정보를 한 번에 보여주려 하지 않습니다.
v0.1에서는 장소 정보, 업체 제공 정보, 방문자 확인 정보, 제보 구조를 위한 기본 UI부터 작게 시작합니다.

군산.com은 Dechive의 검증 철학을 지역 생활 정보에 적용하되, 사용자가 처음 느끼는 인상은 밝고 명확한 지역 정보 검색 사이트여야 합니다.

Dechive처럼 고딕·잡지형 디자인이 아닙니다.
군산.com은 누구나 쉽게 이해하고, 빠르게 찾고, 비교할 수 있는 정보 중심 사이트입니다.

**디자인 키워드:**
명확함 · 신뢰감 · 빠른 탐색 · 쉬운 비교 · 정보 우선 · 광고 느낌 최소화

---

## 디자인 목표

군산.com의 디자인은 다음 질문에 답해야 합니다.

- 여기가 군산 정보를 찾는 사이트라는 것이 바로 보이는가?
- 사용자가 원하는 정보를 빠르게 검색할 수 있는가?
- 장소 정보와 검증 상태가 한눈에 보이는가?
- 업체 제공 정보와 방문자 확인 정보가 구분되는가?
- 광고 사이트처럼 보이지 않는가?
- 데스크탑에서 넓게 비교하고 탐색할 수 있는가?
- 모바일에서도 정보가 깨지지 않고 자연스럽게 쌓이는가?

## 핵심 브랜드 문장

군산.com의 가장 큰 브랜드 방향은 다음 문장입니다.

> 군산.com = 군산의 모든 것

사용자에게 보이는 메인 메시지는 쉽고 단순해야 합니다.

### 메인 카피

| 구역             | 문구                                                                           |
| ---------------- | ------------------------------------------------------------------------------ |
| 큰 문구          | 군산의 모든 것, 군산.com에서 확인하세요                                        |
| 작은 문구        | 맛집부터 생활업체, 여행, 행사까지 군산 정보를 한곳에 모읍니다.                 |
| 검증 보조 문구   | 업체 제공 정보와 방문자 확인 정보를 구분해 더 정확한 군산 정보를 만들어갑니다. |
| 검색 placeholder | 남자 네일, 일요일 병원, 주차 카페                                              |

### 사용 가능한 보조 문장

- 군산 여행 가고 싶어? 군산.com에서 찾아보세요.
- 군산에서 어디 갈지 고민될 때, 군산.com.
- 군산의 맛집, 생활업체, 행사 정보를 한곳에서 확인하세요.
- 업체 제공 정보와 방문자 확인 정보를 나누어 보여드립니다.
- 정보가 달라졌다면 제보해주세요.

## 레이아웃 원칙

군산.com은 데스크탑 정보 탐색을 기본으로 합니다.

모바일에서도 잘 보여야 하지만, 모바일 앱처럼 보이는 구조를 우선하지 않습니다.
기본은 웹사이트이며, 모바일은 반응형으로 자연스럽게 대응합니다.

### 기본 원칙

- 기본: 데스크탑 정보 탐색 최적화
- 반응형: 모바일에서도 보기 좋게 대응
- 앱처럼 보이는 모바일 전용 UI 금지
- 정보 비교가 쉬운 카드형 레이아웃 사용
- 검색, 필터, 장소 목록, 보조 정보를 명확히 분리

---

## 데스크탑 3단 레이아웃

데스크탑에서는 다음 3단 구조를 기본으로 합니다.

```txt
[좌: 필터 패널] | [중: 장소 목록] | [우: 사이드바 / 미리보기]
```

### 좌측 영역

역할:

- 카테고리 필터
- 태그 필터
- 검증 상태 필터
- 동네/지역 필터
- 빠른 조건 필터

예시:

- 맛집
- 카페
- 네일/뷰티
- 병원/약국
- 생활업체
- 주차 가능
- 오늘 영업
- 방문자 사진 있음
- 확인 필요

### 중앙 영역

역할:

- 장소 카드 목록
- 검색 결과
- 추천 장소
- 최근 확인된 장소

장소 카드는 정보 우선으로 구성합니다.

### 우측 영역

역할:

- 선택된 장소 요약
- 자주 찾는 질문
- 최근 확인 정보
- 정보 제보 CTA
- 군산.com의 검증 원칙 안내

---

## 모바일 레이아웃 원칙

모바일에서는 데스크탑 3단 구조를 억지로 유지하지 않습니다.

다음 순서로 자연스럽게 쌓습니다.

1. 헤더
2. 검색 영역
3. 빠른 질문 칩
4. 카테고리 탭
5. 필터 토글
6. 장소 카드 목록
7. 보조 정보
8. 제보 CTA

### 모바일 금지 사항

- 앱처럼 보이는 하단 탭바
- 지나치게 큰 CTA 버튼 반복
- 화면을 가리는 과한 모달
- 검색보다 이미지가 먼저 보이는 구조
- 정보보다 감성 문구가 더 커지는 구조

## 색상 시스템

군산.com은 밝고 명확해야 합니다.

| 용도                  | 방향                                                |
| --------------------- | --------------------------------------------------- |
| 전체 배경             | 흰색 또는 아주 연한 회색 (`slate-50` 수준)          |
| 카드 배경             | 흰색                                                |
| 본문 텍스트           | 진한 slate 계열 (`slate-800` ~ `slate-900`)         |
| 보조 텍스트           | `slate-500` ~ `slate-600`                           |
| 경계선                | `slate-200` 수준                                    |
| 포인트 컬러           | 군산 바다 느낌의 파랑/청록 (`sky-500` ~ `teal-500`) |
| 확인 상태 배지        | 초록 (`green-500` ~ `emerald-500`)                  |
| 미확인 / 확인 필요    | 회색 또는 노랑 (`slate-400`, `yellow-400`)          |
| 현장 차이 제보 있음   | 주황 (`orange-500`)                                 |
| 폐업/이전 확인 필요   | 빨강 (`red-500`)                                    |
| 업체 제공 정보 영역   | 파랑 계열 구분선 또는 배경 틴트                     |
| 방문자 확인 정보 영역 | 초록 계열 구분선 또는 배경 틴트                     |

구체적인 색상 값은 Tailwind CSS 유틸리티로 구현합니다.

디자인 토큰이 필요할 경우 추후 `tailwind.config.ts`에 추가합니다.

## 검증 상태 배지 (StatusBadge)

`VerificationStatus` 값에 따라 색상과 레이블을 다르게 표시합니다.

| 상태                | 배지 색      | 의미                                |
| ------------------- | ------------ | ----------------------------------- |
| 최근 확인됨         | 초록         | 방문자 또는 관리자가 최근 직접 확인 |
| 방문자 후기         | 초록 연한 톤 | 방문자 후기 존재                    |
| 방문자 사진         | 초록 연한 톤 | 방문자 사진 존재                    |
| 업체 제공           | 파랑         | 사장님 제공 정보                    |
| 정보 수집됨         | 회색         | 기본 정보만 있음                    |
| 확인 필요           | 노랑         | 정보 최신성 불확실                  |
| 현장 차이 제보 있음 | 주황         | 온라인 정보와 실제 차이 제보 있음   |
| 폐업/이전 확인 필요 | 빨강         | 폐업 또는 이전 가능성               |

### 배지 디자인 원칙

- 너무 화려하게 만들지 않습니다.
- 상태가 한눈에 보이게 합니다.
- 색상만으로 의미를 전달하지 않고 텍스트를 함께 표시합니다.
- “최근 확인됨”과 “업체 제공”을 절대 같은 의미처럼 보이게 하지 않습니다.
- 확인되지 않은 정보를 긍정적으로 보이게 꾸미지 않습니다.

## 타이포그래피

한글 가독성을 최우선으로 합니다.

### 기본 원칙

- 제목: 굵고 명확하게
- 본문: 읽기 쉬운 중간 굵기
- 정보성 텍스트: 작더라도 판독 가능한 크기 유지
- 최소 본문 크기: `text-sm`
- 과도한 이탤릭체 금지
- 감성적 손글씨 서체 금지
- 장식용 폰트 남용 금지

### 텍스트 계층

| 역할        | 방향                    |
| ----------- | ----------------------- |
| 페이지 제목 | 크고 명확하게           |
| 섹션 제목   | 굵고 짧게               |
| 장소명      | 카드에서 가장 잘 보이게 |
| 태그/상태   | 작지만 명확하게         |
| 설명 문구   | 짧고 읽기 쉽게          |
| 보조 정보   | 회색 톤으로 낮은 위계   |

## 컴포넌트 디자인 규칙

## Header

### 역할

군산.com의 첫 진입점입니다.

### 포함 요소

- `군산.com` 로고
- 검색창
- 장소 둘러보기 링크
- 제보하기 링크
- 사장님 등록 링크

### 디자인 원칙

- 밝고 단정해야 합니다.
- 검색창이 잘 보여야 합니다.
- 사장님 등록 버튼은 보이되 광고처럼 과하게 강조하지 않습니다.
- 데스크탑에서는 한 줄 배치가 기본입니다.
- 모바일에서는 자연스럽게 줄바꿈되거나 검색창이 아래로 내려갈 수 있습니다.

## SearchHero

### 역할

메인 페이지에서 군산.com의 정체성을 가장 먼저 전달합니다.

### 포함 요소

- 큰 문구
- 작은 문구
- 검증 보조 문구
- 검색창 또는 검색 진입 UI
- CTA 버튼

### 디자인 원칙

- 검색창이 가장 눈에 띄어야 합니다.
- 배경을 지나치게 화려하게 꾸미지 않습니다.
- “군산의 모든 것”이라는 인상이 바로 전달되어야 합니다.
- 과한 그라디언트나 SaaS 랜딩 페이지 느낌을 피합니다.

## QuickQuestionChips

### 역할

사용자가 자주 묻는 질문을 빠르게 누를 수 있게 합니다.

### 기본 칩 목록

- 남자 네일 가능한 곳
- 오늘 여는 곳
- 주차 가능한 곳
- 아이랑 가기 좋은 곳
- 혼밥 가능한 곳
- 방문자 사진 있는 곳

### 디자인 원칙

- 작고 명확한 pill 형태
- 클릭 가능한 버튼처럼 보여야 함
- 실제 질문형 문구 사용
- 모바일에서는 가로 스크롤 또는 자연스러운 줄바꿈 허용
- 광고 버튼처럼 과하게 강조하지 않음

## PlaceCard

### 역할

장소 정보를 빠르게 비교하게 합니다.

### 반드시 포함해야 하는 정보

- 장소명
- 동네
- 카테고리
- 핵심 태그
- 상태 배지
- 후기 수
- 최근 확인일
- 상세 보기 링크

### 디자인 원칙

- 장소명은 크고 굵게 표시합니다.

- 후기 수와 최근 확인일은 작게 표시합니다.
- 카드 전체가 클릭 가능한 링크여도 됩니다.
- 광고처럼 보이는 강조 효과는 금지합니다.
- 별점 5개 중심 UI는 사용하지 않습니다.

## RightSidebar

### 역할

장소 탐색 중 보조 정보를 제공합니다.

### 포함 요소

- 자주 찾는 질문

### 예시 문구

- 사이드바는 보조 역할입니다.
- 중앙 장소 목록보다 더 튀면 안 됩니다.
- CTA는 명확하지만 과하면 안 됩니다.

## 장소 상세 페이지 디자인 원칙

장소 상세 페이지는 블로그 글처럼 길게 쓰지 않습니다.

### 우선순위

1. 장소명
2. 카테고리 / 동네
3. 검증 상태
4. 핵심 체크 정보
5. 업체 제공 정보
6. 방문자 확인 정보
7. 사진 출처 구분
8. 정보 차이 제보 CTA

### 금지

- 블로그형 긴 소개글
- 광고 문구 중심 구성
- 출처가 섞인 사진 갤러리
- 확인되지 않은 정보를 확정처럼 표현
- 업체 제공 정보와 방문자 확인 정보 혼합

## 정보 분리 원칙

업체 제공 정보와 방문자 확인 정보는 UI에서 반드시 시각적으로 구분합니다.

### 업체 제공 정보

표시 예시:

- 섹션 제목: `업체 제공 정보`
- 색상 방향: 파랑 계열 구분선 또는 배경 틴트
- 의미: 사장님 또는 업체가 제공한 정보

### 방문자 확인 정보

표시 예시:

- 섹션 제목: `방문자 확인 정보`
- 색상 방향: 초록 계열 구분선 또는 배경 틴트
- 의미: 실제 방문자 제보 또는 확인 정보

### 확인 필요 정보

표시 예시:

- 레이블: `확인 필요`
- 색상 방향: 노랑 또는 회색
- 의미: 최신성 또는 정확성이 아직 불확실한 정보

### 금지

- 하나의 카드나 섹션 안에 출처가 다른 정보를 섞지 않습니다.
- 업체 제공 사진을 방문자 사진처럼 보이게 하지 않습니다.
- 방문자 확인이 없는 정보를 최근 확인된 것처럼 표시하지 않습니다.

## 광고·후원 UI 원칙

군산.com은 광고판처럼 보이면 안 됩니다.

### 허용

- 후원 안내
- 운영 응원 문구
- 사장님 등록 안내
- 업체 제공 정보 표시

### 금지

- 광고비를 낸 장소를 검증된 장소처럼 표시
- 광고비를 낸 장소를 무조건 상단 추천
- 검증 배지와 광고 배지 혼동
- 배너 광고 중심 레이아웃
- 과도한 CTA 반복

### 후원 문구 예시

- 군산.com이 도움이 됐다면 커피 한 잔으로 응원해주세요.
- 더 정확한 군산 정보를 만들 수 있도록 힘을 보태주세요.

## 최종 디자인 기준

군산.com은 예쁜 사이트보다 **쓸모 있는 사이트**여야 합니다.

사용자가 들어왔을 때 가장 먼저 느껴야 하는 것은 다음입니다.

- 군산 정보를 찾는 곳이구나
- 검색하기 쉽구나
- 장소를 비교하기 쉽구나
- 정보 상태가 보이는구나
- 업체 말과 방문자 확인이 구분되는구나
- 내가 아는 정보가 틀리면 제보할 수 있구나

군산.com의 큰 비전은 “군산의 모든 것”입니다.
하지만 디자인은 항상 정보 우선, 신뢰 우선, 빠른 탐색 우선으로 구성합니다.
